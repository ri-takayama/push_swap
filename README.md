## 💡 プロジェクト概要
push_swapは、与えられた整数のリストを2つのスタック（スタックAとスタックB）および特定の操作（命令）のみを使用してソートするプログラムを作成する課題です。 <br>
目標は、可能な限り少ない操作回数でソートを完了させることであり、アルゴリズムの効率性が問われる最適化パズルです。
## 🗣️ 使用可能な命令
このプロジェクトでは、以下の11種類の操作のみを使用できます。
<table>
  <tbody>
    <tr>
      <td>sa / sb / ss</td>
      <td>スタックA / スタックB / 両方のスタックの先頭2つの要素をスワップ（交換）します。</td>
    </tr>
    <tr>
      <td>pa / pb</td>
      <td>スタックA / スタックBの先頭要素を、もう一方のスタックの先頭にプッシュ（移動）します。</td>
    </tr>
    <tr>
      <td>ra / rb / rr</td>
      <td>スタックA / スタックB / 両方のスタックの要素を1つ上にローテート（回転）させます（先頭の要素が末尾に移動）。</td>
    </tr>
    <tr>
      <td>rra / rrb / rrr</td>
      <td>スタックA / スタックB / 両方のスタックの要素を1つ下にリバースローテート（逆回転）させます（末尾の要素が先頭に移動）。</td>
    </tr>
  </tbody>
</table>

## 🧠 使用したアルゴリズム
### 【タークアルゴリズム (Turk Algorithm)】 <br>
本プロジェクトでは、効率的なソートを達成するためにタークアルゴリズムに基づいた戦略を採用しました。 <br>
このアルゴリズムは、スタックAからスタックBに要素を移動させる際、Bスタック内でターゲット要素を最適な位置に配置するための最少のローテーション（回転）操作を計算します。その後、Aスタックに戻す際も、同様に最少の手数で挿入できる位置を特定し、全体の操作回数を最小化することを目指します。 <br>
<br>

## 🛠️ ビルドと実行方法
1. ビルド <br>
リポジトリをクローンした後、`make`コマンドで実行ファイル`push_swap`を生成します。
```bash
git clone git@github.com:ri-takayama/push_swap.git
cd push_swap
make
```
2. 実行 <br>
ソートしたい整数(重複なし)のリストを引数として渡します。プログラムは、ソートに必要な命令を出力します。
```bash
# 実行
./push_swap 2 1 3 6 5 8

# 出力例（ソートに必要な命令列）:
# ra
# pb
# ra
# pa
# ...
```
3. (補足) <br>
42Tokyoの課題には、出力された命令列が実際にリストをソートするかを確認するためのcheckerプログラムが提供されます。
```bash
# 実行例: 
# ( ./push_swap で生成した命令を checker にパイプで渡す )
ARG="2 1 3 6 5 8";./push_swap $ARG | ./checker $ARG
```
